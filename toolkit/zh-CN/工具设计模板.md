# 工具设计模板

构建智能体可调用工具的实用模板。每个工具都是智能体（调用者）与能力（被调用者）之间的契约。正确设计这个契约是智能体系统中最重要的设计决策。

## 最小工具结构

每个工具必须实现五个核心要素：

```typescript
Tool {
  name: string              // 唯一标识符，动词+名词模式
  inputSchema: JSONSchema   // 参数的严格 Schema
  call(input): Result       // 执行操作，返回结构化输出
  prompt(): string          // 系统提示词片段：何时/如何使用
  render(result): string    // 结果的人类可读格式化
}
```

**name** -- 简短，推荐动词+名词模式。`QueryDatabase`、`ReadFile`、`SendEmail`。名称出现在模型的工具列表中，必须不言自明。

**inputSchema** -- JSON Schema，明确列出 `required` 字段。每个属性都包含 `description`。严格约束类型：值有限时用 `enum`，字符串用 `pattern`，数字用 `minimum`/`maximum`。

**call(input)** -- 执行主体。尽可能保持幂等。返回结构化结果对象，而非原始字符串。抛出带可操作信息的类型化错误。

**prompt()** -- 注入系统提示词。告诉模型*何时*适合使用此工具、*做什么*以及*不做什么*。控制在 200 词以内。

**render(result)** -- 将结果转换为模型可推理的字符串。去除不必要的细节。对大输出进行截断并附摘要行。

## 逐步检查清单

1. **定义能力边界。** 此工具做什么？明确不做什么？各写一句话。
2. **起草输入 Schema。** 列出每个参数。逐一决定：必需还是可选？什么类型？什么约束？
3. **实现 call()。** 优雅处理错误。返回结构化数据。记录日志以便观测。
4. **编写 prompt()。** 用模型能理解的术语描述工具。内联 1-2 个使用示例。
5. **实现 render()。** 为模型消费格式化输出。优先用表格或键值对，而非散文。
6. **添加验证。** 执行前验证输入。Schema 违规时返回清晰的错误信息。
7. **用模型测试。** 运行 10 个不同的提示。检查：模型是否正确选择此工具？参数是否正确？是否被误用？
8. **迭代 prompt()。** 大多数工具质量问题出在这里。持续优化直到模型选择准确率超过 95%。

## 可选方法决策矩阵

| 方法 | 何时实现 | 何时跳过 |
|---|---|---|
| `userConfirm(input)` | 工具有副作用（写入、删除、发送） | 只读工具 |
| `estimateCost(input)` | 工具调用付费 API 或长时间运行的进程 | 快速、免费的操作 |
| `validatePermission(input)` | 工具访问敏感资源 | 公共/沙箱化资源 |
| `cacheKey(input)` | 相同输入始终产生相同结果 | 时间敏感或有状态的结果 |
| `timeout()` | 工具可能挂起（网络调用、Shell 命令） | 内存操作 |
| `rollback(result)` | 工具修改可能需要回滚的状态 | 幂等或只读工具 |
| `isReadOnly()` | 工具从不写入磁盘或网络 | 任何有副作用的工具 |
| `isConcurrencySafe()` | 工具可以安全地与自身并行运行 | 修改共享状态的工具 |
| `isDestructive()` | 工具执行不可逆操作 | 可以撤销的工具 |
| `progressCallback()` | 工具运行超过 2 秒 | 亚秒级工具 |

## 关键原则

- **大声失败。** 静默返回空结果的工具比抛出错误的工具更糟糕。模型无法调试沉默。
- **约束输入。** Schema 越严格，模型幻觉参数越少。积极使用 enum、范围和 pattern。
- **提示词为王。** 模型根据 prompt() 而非实现来决定是否使用工具。将 80% 的迭代时间花在这里。
- **为推理而渲染。** 模型将对 render() 输出进行推理。据此目的来结构化——表格优于段落，计数优于列表。
- **一工具一职责。** 如果你的工具做两件事，就拆分它。`ReadFile` 和 `WriteFile`，而不是 `FileOperation`。
- **默认幂等。** 如果模型重试工具调用（这会发生），第二次调用不应损坏状态。
