# Agent 生成模板

设计 Agent 生成系统的实用模板。生成是父 Agent 创建子 Agent 以分配工作的方式。生成模式决定了隔离、通信和生命周期特性。

## Agent 定义格式

每个 Agent 需要一个定义来描述其身份、能力和约束。

```typescript
type AgentDefinition = {
  agentType: string                // 唯一名称，如 "code-reviewer"
  whenToUse: string                // 用于父模型工具提示的描述
  source: 'built-in' | 'user' | 'project' | 'plugin'
  tools?: string[]                 // 工具白名单
  disallowedTools?: string[]       // 工具黑名单
  model?: string                   // 模型覆盖或 'inherit'
  maxTurns?: number                // 最大对话轮数
  permissionMode?: 'default' | 'plan' | 'bubble'
  getSystemPrompt(): string        // Agent 系统提示词
  isolation?: 'worktree'           // Git worktree 隔离
}
```

## 生成模式

### 模式 1：前台（同步）

- 父级阻塞直到 Agent 完成。共享中断控制器。权限提示显示在父终端。
- **适用场景：** 下一步需要结果时。任务是依赖项。

### 模式 2：后台（异步）

- 父级立即继续。Agent 获得独立的中断控制器。权限提示自动拒绝。结果通过通知消息传递。
- **适用场景：** 有真正独立的并行工作时。测试、构建、代码检查。

### 模式 3：分叉（上下文继承）

- 继承父级的完整对话历史。共享提示缓存（成本高效）。始终在后台运行。必须使用与父级相同的模型。
- **适用场景：** 需要父级已知所有内容的研究任务，但中间工作不应污染父级上下文。

## 上下文继承矩阵

| 组件 | 前台 | 后台 | 分叉 |
|------|------|------|------|
| 对话消息 | 全新 | 全新 | 完整副本 |
| 系统提示 | Agent 自己的 | Agent 自己的 | 父级的精确字节 |
| 中断控制器 | 共享 | 独立 | 独立 |
| 文件读取缓存 | 全新 | 全新 | 从父级克隆 |
| 权限上下文 | 继承 | 继承 + 自动拒绝 | 继承 + 冒泡 |

## 通信模式

- **通知：** 后台/分叉 Agent 完成时注入 `<task-notification>` XML 消息到父级对话。
- **进度回调：** 执行期间通过 `onProgress` 报告进度。
- **SendMessage：** Agent 完成后可发送新指令继续。
- **邮箱：** 群体/团队场景中的点对点通信，基于文件系统。

## 决策树：选择哪种模式？

```
下一步之前需要结果吗？
├── 是 --> 前台（阻塞父级，结果直接可用）
└── 否 --> Agent 需要父级的完整对话上下文吗？
           ├── 是 --> 分叉（继承一切，共享缓存，必须同模型）
           └── 否 --> 后台（全新上下文，独立生命周期，可用不同模型）
```

## 生命周期清理

Agent 启动期间分配的每个资源都必须在 `finally` 块中释放：MCP 服务器断开、hooks 注销、文件缓存清除、消息数组释放、孤立 shell 任务终止、状态条目移除。忘记清理会导致内存泄漏和孤立进程。
