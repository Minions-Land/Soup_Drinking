# 第一章：理解 Claude Code 的架构 -- 全景概览

欢迎来到本教程系列。我们将以 Claude Code 的源代码为参考架构，学习如何构建智能体（Agent）系统。无论你是在构建自己的 AI 命令行工具、自主编码助手，还是任何智能体系统，理解 Claude Code 的结构都会为你打下坚实基础。

## Claude Code 是什么？

Claude Code 是一款面向软件工程的 AI 命令行工具。你输入自然语言请求，它就会自主地读取文件、编写代码、执行命令、搜索代码库，甚至协调子智能体。可以把它想象成一位住在你终端里的资深开发者。

但在友好的界面之下，是一套精心设计的智能体架构。让我们揭开帷幕。

## 三大支柱

Claude Code 的架构建立在三个核心抽象之上：

| 支柱 | 角色 | 关键文件 |
|------|------|----------|
| **QueryEngine（查询引擎）** | 大脑 -- 管理对话状态、调用模型、循环直到完成 | `src/QueryEngine.ts` |
| **Tools（工具）** | 双手 -- 模型可以调用的离散能力（读文件、执行命令等） | `src/Tool.ts` + `src/tools/` |
| **Tasks（任务）** | 工人 -- 后台进程和长时间运行的操作 | `src/Task.ts` + `src/tasks/` |

换个方式理解：**QueryEngine** 决定"做什么"，**Tools** 决定"怎么做"，**Tasks** 处理与主对话"并行运行"的工作。

## 数据流：从按键到结果

以下是用户请求在系统中的完整旅程：

```
 用户输入
     |
     v
 +-----------------------+
 | 命令解析               |  commands.ts / processUserInput()
 | （斜杠命令等）          |
 +---------+-------------+
           |
           v
 +-----------------------+
 | QueryEngine           |  QueryEngine.ts
 | - 构建系统提示词        |
 | - 调用 Claude API      |
 | - 进入智能体循环        |
 +---------+-------------+
           |
           v
 +-----------------------+
 | query() 循环           |  query.ts
 | - 流式接收响应（💡 流式输出/Streaming：像打字机一样逐字显示，而不是等全部生成完再一次性显示）          |
 | - 检测 tool_use        |
 | - 执行工具             |
 | - 将结果反馈给模型      |
 +---------+-------------+
           |
       tool_use?
      /        \
    是           否
     |           |
     v           v
 +----------+  +----------+
 | 工具       |  | 渲染      |
 | 执行管道    |  | 最终响应   |
 +----------+  +----------+
     |
     v
 结果反馈到
 query() 循环
 （下一轮 API 调用）
```

这是一个 **ReAct 循环**（💡 即"推理+行动"循环：AI想一步，做一步，看结果，再继续——类似"想→做→看结果→再想→再做"的节奏）。模型推理该做什么，通过调用工具来行动，观察结果，然后重复直到有最终答案。循环位于 `query.ts` 中，运行直到模型发出 `stop_reason: "end_turn"`。

让我们逐步走过每个环节：

**1. 命令解析** -- 用户的文本由 `processUserInput()` 处理。斜杠命令（如 `/compact` 或 `/help`）在本地处理，不会发送到模型。其他内容都变成用户消息。

**2. QueryEngine** -- `QueryEngine` 类（在 `src/QueryEngine.ts` 中）拥有整个对话的生命周期。它组装系统提示词，注入用户上下文（工作目录、项目信息、工具描述），然后启动查询循环。一个 `QueryEngine` 实例贯穿整个对话会话。

**3. query() 循环** -- `src/query.ts` 中的 `query()` 函数是智能体循环的核心。它调用 Claude API，流式接收响应，当模型发出 `tool_use` 块时，执行对应的工具。工具的结果作为 `tool_result` 消息追加，循环继续。这个过程反复进行，直到模型发出最终文本响应，`stop_reason` 为 `"end_turn"`。

**4. 工具执行** -- 每个工具经历一个验证管道：模式验证（Zod，💡 一个TypeScript校验库，用来检查数据格式是否正确，如 `z.string()` 验证某值是字符串）、`validateInput()`、`checkPermissions()`、然后 `call()`。我们将在第二章详细介绍。

**5. 结果渲染** -- 最终的助手消息通过 Ink（CLI 的 React 框架）渲染到终端。每个工具也有自己的渲染方法来展示进度和结果。

## 目录结构概览

```
src/
 |-- QueryEngine.ts      # 大脑：对话生命周期
 |-- Tool.ts              # 工具接口 + buildTool() 工厂函数
 |-- Task.ts              # 任务类型 + 状态管理
 |-- commands.ts          # 斜杠命令注册表
 |-- query.ts             # 智能体循环（模型调用 + 工具分发）
 |-- tools.ts             # 工具池组装
 |
 |-- tools/               # 每个工具一个目录
 |   |-- BashTool/        # Shell 命令执行
 |   |-- FileReadTool/    # 带行范围的文件读取
 |   |-- FileEditTool/    # 精确的文件编辑
 |   |-- FileWriteTool/   # 完整的文件写入
 |   |-- GrepTool/        # 跨文件正则搜索
 |   |-- GlobTool/        # 文件模式匹配
 |   |-- AgentTool/       # 子智能体生成（第三章）
 |   |-- SendMessageTool/ # 智能体间通信
 |   +-- ...              # 30+ 更多工具
 |
 |-- tasks/               # 后台任务实现
 |-- commands/            # 斜杠命令实现
 |-- ink/                 # 终端 UI 组件（React/Ink）
 |-- bridge/              # IDE 集成桥接
 |-- services/            # API 客户端、MCP（💡 Model Context Protocol，让AI工具和外部服务标准化通信的协议，类似USB接口的统一标准）、分析
 |-- utils/               # 共享工具函数（权限、配置等）
 |-- context/             # 系统提示词上下文组装
 +-- state/               # 应用状态（AppState）
```

## 首先阅读的关键文件

如果你想通过阅读源代码来理解 Claude Code，请从以下五个文件开始：

1. **`src/Tool.ts`** -- 定义 `Tool` 接口和 `buildTool()` 工厂函数。每个工具都符合这个形状。
2. **`src/tools/FileReadTool/FileReadTool.ts`** -- 一个结构良好的工具。阅读它可以学到完整的工具生命周期。
3. **`src/QueryEngine.ts`** -- 核心循环。跟踪 `submitMessage()` 看模型如何被调用以及工具调用如何被分发。
4. **`src/Task.ts`** -- 任务抽象。理解后台工作如何被调度和跟踪。
5. **`src/tools/AgentTool/AgentTool.tsx`** -- 多智能体入口。看子智能体如何以自己的上下文被生成。

## 设计哲学

Claude Code 体现了几个值得学习的原则：

- **工具是数据，不是代码。** 每个工具都声明为数据结构（名称、描述、模式、处理器）。系统可以在不执行工具的情况下进行内省、过滤和组合。
- **模型是编排器。** 代码不硬编码工作流。它给模型一组工具，让模型决定执行顺序。
- **安全是分层的。** 权限、沙箱、钩子和验证各自提供独立的防御层。
- **UI 是解耦的。** 终端渲染器是一个 React 应用（通过 Ink），订阅状态变化。核心逻辑对渲染一无所知。
