## 1. Tool 接口

每个工具都实现此泛型接口：

```typescript
type Tool<Input, Output, Progress> = {
  name: string                    // 唯一标识符
  aliases?: string[]              // 向后兼容别名
  searchHint?: string             // ToolSearch 关键词匹配
  shouldDefer?: boolean           // 通过 ToolSearch 延迟加载
  maxResultSizeChars: number      // 自动持久化阈值

  // Schema 定义
  inputSchema: ZodSchema          // Zod v4 输入验证
  outputSchema?: ZodSchema        // 可选输出验证
  inputJSONSchema?: object        // MCP 工具用（原始 JSON Schema）

  // 核心生命周期
  call(args, context, canUseTool, parentMessage, onProgress?)
  validateInput?(input, context) -> ValidationResult    // 逻辑验证
  checkPermissions(input, context) -> PermissionResult   // 权限检查
  description(input, options) -> string

  // 能力声明
  isEnabled() -> boolean           // 是否启用
  isReadOnly(input) -> boolean     // 是否只读
  isDestructive?(input) -> boolean // 是否有破坏性
  isConcurrencySafe(input) -> boolean // 是否并发安全
  interruptBehavior?() -> 'cancel' | 'block' // 中断行为

  // UI 渲染（React/Ink）
  renderToolUseMessage(input, options) -> ReactNode       // 工具调用展示
  renderToolResultMessage?(content, progress, options)    // 工具结果展示
  renderToolUseProgressMessage?(progress, options)        // 进度展示
  renderGroupedToolUse?(toolUses, options)                // 分组展示

  // 工具函数
  prompt(options) -> string       // 该工具的系统提示词
  userFacingName(input) -> string // 用户可见名称
  getPath?(input) -> string       // 获取文件路径
  toAutoClassifierInput(input)    // 安全分类器输入
  checkPermissions(input, context) // 权限检查
}
```

## 2. buildTool 辅助函数

所有工具通过 `buildTool()` 创建，自动填充安全默认值：

```typescript
const MyTool = buildTool({
  name: 'MyTool',
  inputSchema: z.object({ ... }),
  call: async (args, context) => { ... },
  // buildTool 自动提供的默认值：
  // isEnabled: () => true          — 默认启用
  // isConcurrencySafe: () => false — 默认非并发安全
  // isReadOnly: () => false        — 默认可写
  // isDestructive: () => false     — 默认非破坏性
  // checkPermissions: () => allow  — 默认允许
  // toAutoClassifierInput: () => '' — 默认跳过安全分类
})
```

## 3. ToolUseContext（工具上下文）

传递给每个工具调用的丰富上下文：

```typescript
type ToolUseContext = {
  options: {
    commands: Command[]              // 可用命令
    tools: Tools                     // 可用工具
    mcpClients: MCPServerConnection[] // MCP 服务器连接
    mainLoopModel: string            // 当前模型
    thinkingConfig: ThinkingConfig   // 思考配置
    agentDefinitions: AgentDefinitionsResult // 智能体定义
    maxBudgetUsd?: number            // 预算限制
  }
  abortController: AbortController   // 中止控制器
  readFileState: FileStateCache      // 文件读取缓存/去重
  messages: Message[]                // 对话历史
  getAppState(): AppState            // 全局状态
  setAppState(f): void               // 状态变更
  agentId?: AgentId                  // 子智能体 ID
  agentType?: string                 // 智能体类型名
  renderedSystemPrompt?: SystemPrompt // 用于 Fork 缓存共享
}
```

## 4. 两阶段权限系统

工具执行前的双重验证：

```
阶段 1：validateInput(input, context)
   → 逻辑有效性检查（文件是否存在、路径是否合法等）
   → 返回 { result: true } 或 { result: false, message, errorCode }

阶段 2：checkPermissions(input, context)
   → 用户授权检查
   → 返回 { behavior: 'allow' | 'deny' | 'ask', updatedInput }
   → 集成设置规则（alwaysAllow、alwaysDeny、alwaysAsk）
```

权限模式：`default`（默认）、`plan`（计划）、`auto`（自动）、`bypassPermissions`（绕过）

## 5. 工具结果管理

```typescript
type ToolResult<T> = {
  data: T                           // 工具输出
  newMessages?: Message[]           // 注入系统消息
  contextModifier?: (ctx) => ctx    // 修改下次调用的上下文
  mcpMeta?: { ... }                 // MCP 协议元数据
}
```

**自动持久化**：当 `data` 超过 `maxResultSizeChars` 时，结果保存到临时文件，Claude 收到截断预览 + 文件路径。

## 6. 工具目录结构模式

| 模式 | 示例 | 用途 |
|------|------|------|
| `prompt.ts` | 独立的模板字符串文件 | 该工具的系统提示词 |
| `constants.ts` | `TOOL_NAME`, `MAX_RESULT_SIZE` | 共享常量 |
| `UI.tsx` | React/Ink 组件 | 工具专用 UI 渲染 |
| `utils.ts` | 辅助函数 | 工具专用工具函数 |
| Zod schema | `z.object({ file_path: z.string() })` | 输入验证 |

## 7. 核心设计原则

1. **失败时关闭（Fail-closed）** — 工具默认非并发、非只读、非破坏性
2. **两阶段验证** — 先逻辑检查（validateInput），再权限检查（checkPermissions）
3. **上下文感知渲染** — 工具在详细/精简/转录模式下渲染不同
4. **渐进式结果交付** — 长时间操作通过 Progress 回调逐步报告
5. **延迟加载** — 不常用工具通过 ToolSearch 按需加载
6. **安全分类器输入** — 安全相关工具提供结构化输入给自动模式安全检查
