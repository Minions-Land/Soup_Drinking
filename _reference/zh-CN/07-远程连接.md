## 1. Bridge 架构

"Bridge"连接本地 Claude Code 实例到 Anthropic 的远程控制基础设施（claude.ai/code）。

### 两代架构

| 版本 | 名称 | 流程 |
|------|------|------|
| V1 | 基于环境（旧版） | 注册 → 轮询工作 → 确认 → 心跳 |
| V2 | 无环境（直连） | 直接 `/bridge` 端点 → SSE/POST |

### 核心组件

| 文件 | 用途 |
|------|------|
| `bridgeApi.ts` | 环境与会话 API 的 HTTP 客户端 |
| `bridgeMain.ts` | V1 基于环境的流程 |
| `remoteBridgeCore.ts` | V2 无环境直连流程 |
| `replBridge.ts` | REPL 特定的 Bridge 编排 |
| `bridgeMessaging.ts` | 消息序列化、去重（`BoundedUUIDSet`） |
| `bridgePointer.ts` | 本地文件指针用于会话/环境 ID 恢复 |

## 2. 会话管理

```
会话生命周期：创建（POST /v1/sessions）→ 活跃 → 归档（清理）
```

- **元数据**：项目目录、git 仓库 URL、分支、可读标题
- **恢复**：`bridgePointer` 通过本地持久化会话 ID 支持崩溃恢复
- **永久会话**：可存活重启；"原地重连"到已有环境

## 3. 传输机制

| 传输 | 协议 | 方向 | 用途 |
|------|------|------|------|
| SSE | Server-Sent Events | 服务器→客户端 | V2 读取 |
| WebSocket | WS | 双向 | V1、直连 |
| HTTP POST | HTTP | 客户端→服务器 | V2 写入 |
| Hybrid | WS+POST | 混合 | V1 组合 |

序列号管理（`lastTransportSequenceNum`）用于流恢复。

## 4. 认证与安全

| 层级 | 机制 | 用途 |
|------|------|------|
| OAuth | 系统钥匙串令牌 | CLI 到后端认证 |
| JWT（Worker 令牌） | 短期令牌通过 `/bridge` | V2 会话认证 |
| Work Secrets | 不透明连接字符串 | V1 环境凭证 |
| 可信设备 | `X-Trusted-Device-Token` 头 | 设备验证 |
| 令牌刷新 | `TokenRefreshScheduler` | 主动 JWT 刷新（过期前 5 分钟） |
| 进程安全 | `prctl(PR_SET_DUMPABLE, 0)` | 防止内存窃取（Linux） |

## 5. 上游代理

安全容器（CCR）中的 Claude Code 通过托管 MITM 代理访问外部服务：

```
工具（curl/npm）→ HTTP CONNECT → 本地 Relay
  → WebSocket → Anthropic 服务器
  → TLS MITM → 目标服务
```

| 文件 | 用途 |
|------|------|
| `upstreamproxy.ts` | 容器端代理初始化 |
| `relay.ts` | localhost 上的 TCP 到 WebSocket 中继 |

## 6. 服务器模块

`directConnectManager.ts` — "直连"会话：
- 简化的 WebSocket 通信
- 控制请求处理（权限）
- 中断信号管理

## 7. 会话历史

`sessionHistory.ts`：
- `fetchLatestEvents()` — 从后端获取分页会话事件
- `fetchOlderEvents()` — 历史分页
