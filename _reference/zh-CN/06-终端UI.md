# 终端 UI 框架（Ink）

> 🗺️ **导航**: [← README](../README.md) | [英文版](../en/06-INK-UI.md)  
> 📁 [参考索引](../README.md#参考文档) | [教程](../tutorial/) | [工具包](../toolkit/)

> 源码位置：`src/ink/` — 高性能 React-for-CLI 框架，Ink 的定制化 Fork。

## 1. 系统概览

相比标准 Ink 的核心优化：
- **O(dirty) 渲染** — 仅重新处理修改过的子树
- **无 GC 的 Cell 缓冲区** — 使用 TypedArray（Int32Array）消除 GC 暂停
- **Blitting（位块传送）** — 干净子树的输出直接从上一帧复制
- **同步输出** — DEC 2026 模式防止闪烁
- **高级终端探测** — 无阻塞的能力检测

## 2. 渲染管线

```
输入 → React 状态变更 → Reconciler → DOM 更新 → 脏标记
  → 布局（Yoga）→ 渲染树遍历 → Blit/渲染 → 屏幕缓冲区
  → 新旧 Diff → 最小 ANSI 输出 → 同步写入 stdout
```

### 核心组件

| 组件 | 文件 | 职责 |
|------|------|------|
| 渲染器 | `renderer.ts` | 帧生命周期、双缓冲 |
| 节点渲染 | `render-node-to-output.ts` | 树遍历、Blitting 优化 |
| 输出累加器 | `output.ts` | write/blit/clear/clip 操作 |
| 屏幕缓冲 | `screen.ts` | 底层缓冲（Int32Array 打包 Cell） |
| 优化器 | `optimizer.ts` | 合并/简化终端更新补丁 |
| 终端驱动 | `terminal.ts` | 最终 stdout 写入、同步输出 |

### 打包 Cell 格式

每个 Cell = 2 × Int32 字：
- **字 0**：驻留字符 ID
- **字 1**：打包位字段 — 前景色、背景色、属性（粗体/斜体等）、显示宽度

字符、样式、超链接 URI 使用驻留池（Interning Pool）。

## 10. 交互流程总结

```
1. 用户按键
2. parse-keypress 解码 ANSI/Kitty/SGR
3. Dispatcher 发送到聚焦的 DOMElement（捕获 → 冒泡）
4. React 状态变更
5. Reconciler 修改 DOMElement → 标记"脏"
6. Renderer 触发 Yoga 布局计算脏子树
7. RenderNodeToOutput 遍历树：
   干净节点 → 从旧缓冲区 Blit
   脏节点 → 重新渲染到 Output
8. Output 写入 Screen 类型化数组
9. Terminal 对比新旧屏幕 → 生成最小 ANSI Diff
10. 包裹在同步更新序列（DEC 2026）中 → 写入 stdout
```
