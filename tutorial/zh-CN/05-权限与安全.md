# 第五章：权限与安全

## 为什么权限很重要

Claude Code 是一个可以读取文件、编写代码、执行 shell 命令和浏览网页的自主智能体。没有权限系统，一个被误解的指令就可能删除你的代码库、暴露密钥或运行破坏性命令。权限系统使自主操作值得信赖。

## 权限模型

Claude Code 使用**分层权限模型**。每个工具调用在执行前通过多个检查点：

```
工具调用请求
    |
    v
[1. 模式验证]              -- 输入格式正确吗？
    |
    v
[2. validateInput()]       -- 输入语义有效吗？
    |
    v
[3. checkPermissions()]    -- 用户同意此操作吗？
    |
    v
[4. 钩子：PreToolUse]      -- 自定义钩子允许吗？
    |
    v
[5. 沙箱强制]              -- 操作在沙箱范围内吗？
    |
    v
[6. 工具执行]              -- 最终执行工具
    |
    v
[7. 钩子：PostToolUse]     -- 执行后检查
```

每一层都可以独立阻止调用。纵深防御意味着一层中的错误不会危及安全。

### 权限模式

| 模式 | 行为 |
|------|------|
| `default` | 非只读操作询问用户 |
| `plan` | 仅只读工具，不允许变更 |
| `auto` | 模型引导的审批 + 安全分类器 |
| `bypassPermissions` | 跳过所有提示（危险，需显式选择） |

## checkPermissions()

主要权限关卡。每个工具实现自己的版本，返回三种结果之一：**allow**（允许）、**deny**（拒绝）、**ask**（询问用户）。

## 权限规则

用户在 `.claude/settings.json` 中配置。拒绝规则优先于允许规则。

## BashTool 安全

最危险的工具拥有专门的安全子系统：

- **bashSecurity.ts** -- 基于正则的危险模式命令过滤
- **commandSemantics.ts** -- 将命令分类为读/搜索/列表/写操作
- **destructiveCommandWarning.ts** -- 警告 `rm -rf`、`git reset --hard` 等破坏性命令
- **pathValidation.ts** -- 验证文件路径在允许的目录内
- **shouldUseSandbox.ts** -- 确定是否需要沙箱执行

## 沙箱

在操作系统级别限制文件系统和网络访问。在 macOS 上通过 `sandbox-exec` 机制实现。即使工具有 bug，OS 也能阻止未授权访问。

## 权限冒泡

子智能体需要权限时，请求向上冒泡到用户。已授予的权限向下继承。

## 拒绝追踪

用户拒绝权限时，系统追踪以避免重复询问，并通知模型寻找替代方法。

## 钩子：PreToolUse 和 PostToolUse

钩子是在工具执行前后运行的自定义外部进程，通过 stdin 接收 JSON，可以用任何语言编写。PreToolUse 可以阻止、允许或修改工具调用。PostToolUse 可以记录操作、运行 linter 或发送通知。

## 安全设计原则

1. **默认拒绝。** 没有匹配规则时询问用户。
2. **纵深防御。** 多个独立层各自提供保护。
3. **最小权限。** 工具只获得需要的访问权限。
4. **用户主权。** 用户始终拥有最终决定权。
5. **透明。** 每个权限决定都被记录且可见。
6. **关闭失败。** 新工具默认有更多限制，而不是更少。
