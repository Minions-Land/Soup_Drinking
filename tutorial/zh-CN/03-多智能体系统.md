# 第三章：多智能体系统

## 为什么需要多个智能体？

单次模型调用可以完成很多工作，但某些任务对于一个线性的工具调用序列来说太复杂了。Claude Code 通过多智能体架构解决了这个问题：主智能体可以生成子智能体，每个子智能体都有自己的对话上下文、工具和专注领域。

核心洞察：**智能体就是工具**。`AgentTool` 是一个普通工具，只不过它创建新的 AI 对话。模型决定何时委托，就像它决定何时读取文件一样。

## AgentTool：入口点

多智能体系统从 `AgentTool` 开始，定义在 `src/tools/AgentTool/` 中。当模型决定某个子任务最好由单独的智能体处理时，它用描述工作的提示词调用 `AgentTool`。

**关键文件：**
- `runAgent.ts` -- 核心智能体执行循环
- `forkSubagent.ts` -- 上下文分叉用于 Prompt 缓存（💡 把常用的提示词缓存起来，下次直接复用，省钱省时间）共享
- `builtInAgents.ts` -- 内置智能体类型注册表
- `built-in/` -- 各个智能体定义（探索、计划、验证等）
- `loadAgentsDir.ts` -- 从 `.claude/agents/` 加载自定义智能体

## 智能体如何生成：runAgent()

`runAgent()` 函数负责创建和运行子智能体：

1. **上下文分叉。** 从父级上下文创建新的对话上下文。
2. **工具池选择。** 子智能体接收精选的工具集。通常不能生成进一步的子智能体以防止无限递归。
3. **系统提示词组装。** 子智能体获得根据其角色定制的系统提示词。
4. **查询循环执行。** 子智能体进入自己的 `query()` 循环并运行直到产生最终答案。
5. **结果收集。** 子智能体的最终响应作为工具结果返回给父级。

## 上下文分叉：forkSubagent()

最重要的优化：子智能体继承父级系统提示和对话前缀的**字节精确副本**：

```
父级：[系统提示词][消息1][消息2][消息3]
                          | 分叉点
子级：[系统提示词][消息1][消息2][新任务提示词]
      ^ 共享 Prompt 缓存 -- 零重新分词成本！
```

**继承的内容：** 系统提示词、文件读取时间戳、权限授予、工作目录、取消信号。

**不继承的内容：** 分叉点之后的对话历史、工具结果、进行中状态。

这种隔离至关重要。处理"查找函数 X 的所有用法"的子智能体不应被父级正在进行的重构讨论分散注意力。

## SendMessageTool：智能体间通信

`SendMessageTool` 允许智能体向彼此的"收件箱"发送结构化消息。后台智能体通过注入消息流的 XML 块传递结果。

## 协调器模式：四个阶段

对于复杂的多文件任务，工作流遵循四个阶段：

1. **扇出（研究）。** 并行生成多个研究智能体收集信息。
2. **综合（规划）。** 收集所有结果并综合出计划，写入草稿板。
3. **实现。** 生成实现智能体，每个负责计划的特定部分。
4. **验证。** 生成**全新**智能体（无实现上下文）独立审查，防止确认偏差。

## 内置智能体

| 智能体 | 用途 | 可用工具 |
|-------|------|----------|
| `general-purpose` | 通用研究和多步骤任务 | 所有工具 |
| `Explore` | 快速代码库探索 | 只读工具 |
| `Plan` | 架构和实现规划 | 只读工具 |
| `verification` | 独立代码审查 | 只读工具 |

自定义智能体可以在 `.claude/agents/` 中定义并动态加载。

## 后台智能体

后台智能体在用户继续与主对话交互时异步（💡 不等待某个操作完成就继续做其他事，就像发邮件后不用守着等回复）运行。它们不阻塞主对话，通过任务系统报告进度，完成时通过 `<task-notification>` XML 呈现结果，可以被用户取消。

## 设计要点

1. **委托是一个工具。** 将智能体生成作为常规工具调用保持架构统一。
2. **上下文隔离防止混乱。** 子智能体在专注于特定任务的干净上下文中工作效果最好。
3. **Prompt 缓存共享节省 token。** 字节精确的系统提示词副本消除重新分词成本。

> 💡 **新手提示：什么是 Prompt Cache？**  
> 每次调用AI时都要把系统提示词发给模型，如果系统提示词很长，每次都重新处理很浪费。Prompt Cache 会把这段提示词"记住"，子智能体复用同一份缓存，就像复印文件而不是重新打印。  
> 🔍 想深入了解？[搜索"Prompt Cache 是什么"](https://www.baidu.com/s?wd=Prompt+Cache+是什么)
4. **并行研究，串行实现。** 研究可以安全扇出（只读），实现需要串行化（写操作需要协调）。
5. **全新验证防止偏差。** 验证者不应看到实现过程。
6. **深度限制防止失控递归。** 子智能体通常不能生成进一步的子智能体。
