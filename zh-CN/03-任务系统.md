## 1. 任务抽象

每个长时间运行或可后台化的操作都是一个**任务（Task）**：

```
任务生命周期：pending（等待中）→ running（运行中）→ completed（完成）| failed（失败）| killed（被终止）
```

所有任务注册在全局 `AppState` 中 — 支持 UI 追踪、僵尸保护和会话持久化。

### 任务类型

| 类型 | 文件 | 描述 |
|------|------|------|
| `LocalShellTask` | `tasks/LocalShellTask/` | 后台 Bash 进程 |
| `LocalAgentTask` | `tasks/LocalAgentTask/` | 本地子智能体执行 |
| `RemoteAgentTask` | `tasks/RemoteAgentTask/` | 远程智能体执行 |
| `InProcessTeammateTask` | `tasks/InProcessTeammateTask/` | 同进程 Swarm 成员 |
| `DreamTask` | `tasks/DreamTask/` | 后台"梦境"处理 |
| `LocalMainSessionTask` | `tasks/LocalMainSessionTask.ts` | 主会话追踪 |

## 2. 任务管理工具

```
TaskCreateTool  → 创建后台任务
TaskGetTool     → 获取任务输出
TaskListTool    → 列出所有任务（共享工作队列）
TaskOutputTool  → 流式读取任务输出
TaskStopTool    → 停止/终止任务
TaskUpdateTool  → 更新任务元数据
```

## 3. 后台任务流程

```
1. 模型调用 TaskCreateTool（或带 background 标志的 AgentTool）
2. 创建任务，注册到 AppState
3. 任务异步运行
4. 模型可以：
   a. 通过 TaskGetTool 检查进度
   b. 通过 TaskOutputTool 读取输出（block=true/false）
   c. 通过 TaskStopTool 停止
5. 完成通过 <task-notification> XML 通知
6. 僵尸保护：智能体死亡时 killShellTasksForAgent() 清理进程
```

## 4. 持久化与恢复

- 任务状态持久化到 `.claude/projects/...`
- 支持崩溃/重启后的会话恢复
- `DreamTask` 支持跨会话边界的长时间后台处理

## 5. 上下文隔离

使用 `AsyncLocalStorage` 实现：
- **Agent Context** — 追踪函数调用属于哪个智能体
- **Teammate Context** — 隔离进程内队友
- 无需通过调用栈手动传递上下文
